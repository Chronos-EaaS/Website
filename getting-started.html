--- title: Getting Started ---

This page gives an overview on how to deploy and use Chronos.


Setup Chronos Control

There are different approaches on how to setup Chronos. The easiest and recommend way is to use the install script. The GitHub readme outlines the necessary steps to install Chronos Control using the provided install script. Please make sure to check the requirements section and to install missing software if required.


Integrate Chronos Agent

If you already have an evaluation client for your system you now need to connect it with Chronos Control. In the Chronos terminology, we call the module for doing this communication Chronos Agent. We provide a reference implementation of Chronos Agent in Java. We recommend to use this this implementation for your project because it supports all features.

If you are using a non-jvm language it is possible to write your own Chronos Agent. The whole communication between Chronos Agent and Chronos Control is REST-based which makes it easy to implement. Please get in touch with us if you implemented our own Chronos Agent library consider sharing it.

With the MongoDB Demo Client we provide an example on how to integrate Chronos Agent in an evaluation client.

Get in touch with us

Please let us know if you have troubles by either opening an issue on GitHub or directly via Mail.

Also if we did a good job and you have no issues with Chronos we are very happy to hear from you. Getting feedback on how Chronos improves your workflow motivates us to continuously improve Chronos and to add additional features.
